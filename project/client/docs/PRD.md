# 컬러링 앱 PRD (Product Requirements Document)

## 1. 제품 개요

### 1.1 프로젝트 이름

**디지털 컬러링 앱** (Digital Coloring App)

- 초기 버전: PWA (Progressive Web App)
- 향후 확장: React Native 또는 Flutter 네이티브 앱

### 1.2 프로젝트 목적

사용자들이 **웹과 모바일** 환경에서 직관적이고 창의적인 컬러링 경험을 할 수 있는 앱을 제공하여, 스트레스 해소와 창작 활동을 지원합니다. 특히 시니어 사용자들에게는 두뇌 건강 유지와 여가 활동의 기회를 제공하여 디지털 포용성을 실현합니다.

**MVP 전략**: PWA로 빠르게 출시하여 시장 검증 후, 필요에 따라 네이티브 앱으로 확장합니다.

### 1.3 핵심 가치 제안

- **즉시 사용 가능**: 앱스토어 없이 브라우저에서 바로 시작
- **쉬운 설치**: "홈 화면에 추가" 한 번으로 앱처럼 사용
- 언제 어디서나 손쉽게 즐길 수 있는 컬러링 경험
- 다양한 템플릿과 색상 팔레트 제공
- 완성작 저장 및 공유 기능
- 직관적이고 부드러운 사용자 경험
- **시니어 친화적 설계**: 큰 글씨, 명확한 안내, 쉬운 조작
- **세대 간 소통**: 손자녀와 함께 즐길 수 있는 활동
- **두뇌 건강**: 집중력과 기억력 향상에 도움
- **빠른 업데이트**: 자동 업데이트로 항상 최신 버전 사용

## 2. 타겟 사용자

### 2.1 주요 타겟

- **연령대**: 10대 ~ 60대 (전 연령층)
- **관심사**: 취미 활동, 스트레스 해소, 창작 활동, 두뇌 건강 유지
- **사용 시나리오**: 출퇴근 시간, 여가 시간, 휴식 시간, 가족과의 시간

### 2.2 사용자 페르소나

#### 페르소나 1: 직장인 민지 (28세)

- 출퇴근 시간에 가벼운 취미 활동을 원함
- 스트레스 해소가 필요
- 완성작을 SNS에 공유하고 싶어함

#### 페르소나 2: 중학생 지우 (14세)

- 디지털 아트에 관심이 많음
- 다양한 컬러 조합 실험을 좋아함
- 친구들과 작품을 공유하고 싶어함

#### 페르소나 3: 은퇴자 순자 (62세)

- 여가 시간이 많고 새로운 취미를 찾고 있음
- 손자녀와 함께 즐길 수 있는 활동을 원함
- 두뇌 건강과 치매 예방에 관심이 많음
- 큰 글씨와 직관적인 인터페이스를 선호
- 복잡한 기능보다 단순하고 명확한 사용법을 원함

## 3. 핵심 기능 (Core Features)

### 3.1 MVP 필수 기능

#### 3.1.1 컬러링 캔버스

- **기능**: 터치 기반 영역별 색칠 기능
- **요구사항**:
  - 영역 자동 인식 (flood fill 알고리즘)
  - 실시간 색상 적용
  - 줌인/줌아웃 제스처 지원 (핀치, 더블탭)
  - 실수 복구를 위한 되돌리기(Undo) 기능
  - **시니어 친화 기능**:
    - 터치 영역 시각적 강조 (선택 시 테두리 표시)
    - 잘못된 영역 터치 시 부드러운 피드백
    - 확대 시 자동 중앙 정렬
    - 실수 방지: "정말 색칠하시겠습니까?" 확인 옵션 (설정)

#### 3.1.2 색상 팔레트

- **기능**: 다양한 색상 선택 도구
- **요구사항**:
  - 기본 색상 팔레트 (최소 24색)
  - 커스텀 색상 선택기 (Color Picker)
  - 최근 사용한 색상 히스토리
  - 즐겨찾기 색상 저장
  - **시니어 친화 기능**:
    - 색상 이름 텍스트 표시 (빨강, 파랑 등)
    - 큰 색상 버튼 크기 (최소 60x60pt)
    - 색상 간 충분한 여백

#### 3.1.3 템플릿 라이브러리

- **기능**: 다양한 컬러링 템플릿 제공
- **요구사항**:
  - 카테고리별 분류 (동물, 식물, 만다라, 추상, 전통 문양, 풍경화, 꽃, 새 등)
  - 난이도 표시 (쉬움, 보통, 어려움)
  - 썸네일 미리보기
  - 무한 스크롤 또는 페이지네이션
  - **시니어 선호 카테고리**:
    - 전통 문양 (한국 전통 패턴)
    - 계절 풍경 (사계절 자연)
    - 꽃과 정원 (장미, 국화, 백합 등)
    - 새와 동물 (학, 나비, 고양이 등)
    - 명화 따라 그리기 (반 고흐, 모네 등)

#### 3.1.4 저장 기능

- **기능**: 작업 중인 작품 및 완성작 저장
- **요구사항**:
  - **로컬 저장**: IndexedDB (대용량 이미지 저장)
  - **클라우드 저장**: Supabase Storage (백업 및 동기화)
  - 자동 저장 기능 (30초마다)
  - 저장된 작품 목록 관리
  - 작품 삭제 기능
  - **오프라인 지원**: Service Worker로 오프라인에서도 작업 가능

#### 3.1.5 도움말 및 튜토리얼 (시니어 필수)

- **기능**: 언제든 접근 가능한 사용 가이드
- **요구사항**:
  - 첫 실행 시 상세한 튜토리얼
  - 각 기능별 툴팁 제공
  - 화면별 도움말 버튼 (항상 보이는 위치)
  - 동영상 가이드 또는 애니메이션 설명
  - 글자 크기 조절 기능 (설정 메뉴)

#### 3.1.6 PWA 기능 (MVP 필수)

- **기능**: 웹앱을 네이티브 앱처럼 사용
- **요구사항**:
  - **홈 화면 추가**: "홈 화면에 추가" 안내 및 설치 프롬프트
  - **오프라인 작동**: Service Worker로 기본 기능 오프라인 지원
  - **앱 아이콘**: 다양한 크기의 아이콘 (192x192, 512x512)
  - **스플래시 스크린**: 앱 로딩 시 브랜드 화면
  - **전체 화면 모드**: 브라우저 UI 숨김 (standalone 모드)
  - **푸시 알림**: (선택사항) 새 템플릿 추가 알림

### 3.2 향후 추가 기능 (Post-MVP)

#### 3.2.1 공유 기능

- SNS 공유 (이미지 형태)
- 앱 내 갤러리 공유
- 친구 초대 기능
- **가족 공유**: 손자녀에게 작품 직접 전송

#### 3.2.2 고급 도구

- 그라데이션 도구
- 패턴/텍스처 채우기
- 레이어 기능

#### 3.2.3 소셜 기능

- 좋아요 및 댓글
- 인기 작품 랭킹
- 챌린지/이벤트

#### 3.2.4 시니어 전용 기능

- **두뇌 건강 모드**: 매일 새로운 컬러링 챌린지 제공
- **음성 안내**: 각 기능을 음성으로 설명
- **확대경 모드**: 세밀한 영역 작업 시 자동 확대
- **진행 상황 알림**: "오늘 30분 컬러링 완료!" 등의 격려 메시지
- **프린트 기능**: 완성작을 실제로 인쇄할 수 있는 옵션

#### 3.2.5 수익화

## 4. 기술 스택

### 4.1 프론트엔드

- **프레임워크**: Next.js 14 (App Router)
  - **선택 이유**:
    - Yeon님의 Next.js 전문성 활용으로 빠른 개발 가능
    - 3주 내 MVP 완성 확실성 (하루 1시간, 주말 4시간 = 총 39시간)
    - Claude Code 워크플로우에 최적화 (Vercel 배포 → 브라우저 확인 → 실시간 수정)
    - HTML Canvas API가 익숙하여 러닝커브 없음
    - PWA 변환이 간단하여 모바일에서도 앱처럼 사용 가능
    - 향후 React Native 마이그레이션 시 코드 70% 재사용 가능
- **언어**: TypeScript
- **UI 프레임워크**:
  - Tailwind CSS (스타일링)
  - shadcn/ui (컴포넌트 라이브러리)
  - Radix UI (접근성)
- **Canvas 라이브러리**:
  - HTML Canvas API (기본 렌더링)
  - react-konva (복잡한 캔버스 작업 시)
  - 또는 Fabric.js (고급 기능 필요 시)
- **상태 관리**:
  - Zustand (간단하고 직관적)
  - 또는 React Context + useReducer (초기에는 충분)
- **PWA 지원**:
  - next-pwa (Service Worker, 오프라인 지원)
  - Web App Manifest (홈 화면 추가)
- **제스처 핸들링**:
  - React 기본 이벤트 (onTouchStart, onTouchMove)
  - use-gesture (고급 제스처 필요 시)

### 4.2 백엔드

- **백엔드 서비스**: Supabase (오픈소스 Firebase 대안)
  - **선택 이유**:
    - PostgreSQL 기반으로 복잡한 쿼리 및 관계형 데이터 처리 가능
    - Firebase보다 강력한 데이터베이스 기능 (JOIN, 트랜잭션, 인덱싱)
    - 오픈소스로 벤더 락인 최소화
    - 무료 티어가 넉넉하여 MVP에 적합
    - Row Level Security로 강력한 보안
    - Firebase와 유사한 개발자 경험 (DX)
- **주요 기능**:
  - **Database**: PostgreSQL (템플릿, 사용자 작품, 카테고리 관리)
  - **Authentication**: Supabase Auth (이메일, 소셜 로그인)
  - **Storage**: Supabase Storage (템플릿 이미지, 완성작 이미지)
  - **Realtime**: PostgreSQL Changes (실시간 동기화 - 추후 기능)
  - **Edge Functions**: Deno 기반 서버리스 함수 (이미지 처리 등)
- **CDN**: Cloudflare (이미지 전송 최적화)
- **분석**:
  - Supabase Analytics (기본)
  - 또는 Mixpanel / Amplitude (상세 분석)

### 4.3 개발 도구

- **버전 관리**: Git + GitHub
- **패키지 관리**: npm (또는 pnpm)

### 4.4 데이터베이스 스키마 (Supabase PostgreSQL)

```sql
-- 템플릿 테이블
CREATE TABLE templates (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  title TEXT NOT NULL,
  category_id UUID REFERENCES categories(id),
  difficulty TEXT CHECK (difficulty IN ('easy', 'medium', 'hard')),
  image_url TEXT NOT NULL,
  thumbnail_url TEXT NOT NULL,
  usage_count INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

-- 카테고리 테이블
CREATE TABLE categories (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  name TEXT NOT NULL,
  icon TEXT,
  sort_order INTEGER
);

-- 사용자 작품 테이블
CREATE TABLE user_artworks (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES auth.users(id),
  template_id UUID REFERENCES templates(id),
  colored_data JSONB, -- 색칠된 영역 데이터
  progress INTEGER DEFAULT 0, -- 완성도 (0-100)
  is_completed BOOLEAN DEFAULT FALSE,
  image_url TEXT, -- 완성작 이미지 URL
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Row Level Security 정책
ALTER TABLE user_artworks ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can view their own artworks"
  ON user_artworks FOR SELECT
  USING (auth.uid() = user_id);

CREATE POLICY "Users can insert their own artworks"
  ON user_artworks FOR INSERT
  WITH CHECK (auth.uid() = user_id);
```

### 4.5 기술 스택 선택 근거

#### Next.js + PWA를 선택한 이유

**1. 현실적인 개발 일정 (핵심 이유)**

- 개발 가능 시간: 평일 1시간/일, 주말 4시간/일 = 총 13시간/주
- 3주 목표: 총 39시간
- Next.js: Yeon님이 전문가 수준으로 익숙 → 학습 시간 0시간
- 39시간 안에 완성 확률: **95%** ✅

**2. Claude Code 워크플로우 최적화**

- Vercel 자동 배포: git push → 즉시 배포
- 브라우저에서 실시간 확인
- Claude Code로 URL 보면서 수정
- HMR(Hot Module Replacement)로 즉각 반영
- 개발 속도 극대화

**3. PWA의 실용성**

- 컬러링 앱은 PWA로 충분히 실용적
- Canvas API 완벽 지원
- 오프라인 작동 (Service Worker)
- 홈 화면 추가 → 일반 앱처럼 사용
- 앱스토어 심사 없이 즉시 배포
- 업데이트 자동 (시니어에게 유리)

**4. 시니어 사용자 관점**

- 설치가 간단: "홈 화면에 추가" 한 번
- 앱스토어 계정/다운로드 불필요
- 경로당 워크숍에서 QR 코드 스캔 → 즉시 사용
- 네이티브 앱보다 진입 장벽 낮음

**5. 빠른 검증 및 피드백**

- 3주 안에 실제 동작하는 MVP
- 베타 테스터에게 URL만 공유
- 실시간 사용 데이터 수집
- 빠른 개선 및 배포

**6. 향후 확장 가능성**

- PWA로 충분하면 그대로 유지
- 네이티브 앱 필요시 React Native로 전환 (코드 70% 재사용)
- Flutter 필요시 검증된 기획으로 처음부터 재개발
- 유연한 전략

#### Supabase를 선택한 이유 (동일 유지)

1. **강력한 DB**: PostgreSQL로 복잡한 쿼리와 확장 가능
2. **개발 속도**: Next.js와 완벽한 호환, 빠른 통합
3. **비용 효율**: 무료 티어 넉넉, 사용량 기반 합리적 가격
4. **오픈소스**: 벤더 락인 없음, 필요시 자체 호스팅 가능
5. **보안**: Row Level Security로 강력한 데이터 보호
6. **확장성**: Edge Functions로 향후 AI 기능 추가 용이

#### Canvas 렌더링 성능

- HTML Canvas API는 컬러링 앱에 충분한 성능
- 60fps까지는 어렵지만 30-45fps로도 부드러운 경험 가능
- 최적화 기법: requestAnimationFrame, 오프스크린 캔버스
- 시니어 사용자는 성능보다 사용 편의성이 중요

## 5. 사용자 플로우

### 5.1 신규 사용자 온보딩

```
앱 실행 → 사용자 연령대 선택 (선택사항) → 맞춤형 튜토리얼 화면
→ 템플릿 선택 → 컬러링 시작
```

**시니어 사용자 온보딩 플로우**:

```
앱 실행 → "처음 사용하시나요?" 안내
→ 상세 단계별 튜토리얼 (스킵 가능)
→ 쉬운 템플릿 추천 → 안내와 함께 첫 컬러링
→ 완성 축하 메시지 → 저장 방법 안내
```

### 5.2 기본 컬러링 플로우

```
템플릿 선택 → 캔버스 로드 → 색상 선택 → 영역 터치하여 색칠
→ 작품 완성 → 저장/공유
```

### 5.3 작품 관리 플로우

```
내 작품 목록 → 작품 선택 → 계속 작업 or 공유 or 삭제
```

## 7. 비기능 요구사항

### 7.1 성능

- **PWA 초기 로딩**: 3초 이내 (First Contentful Paint)
- **캔버스 렌더링**: 30-45 FPS 목표 (웹 기반으로 현실적인 목표)
- **터치 반응 속도**: 100ms 이내
- **이미지 저장 시간**: 2초 이내 (IndexedDB)
- **Lighthouse 점수**: 90+ (Performance, Accessibility, Best Practices)
- **시니어 사용자 고려**:
  - 튜토리얼 및 도움말 로딩: 1초 이내
  - 확대/축소 제스처 반응: 즉시 (지연 없음)
  - 색상 선택 피드백: 시각적 강조 (햅틱은 브라우저 제한적)

### 7.2 호환성

- **브라우저**:
  - iOS: Safari 13+ (iOS 13+)
  - Android: Chrome 80+, Samsung Internet 12+
  - 데스크톱: Chrome, Edge, Safari (참고용)
- **PWA 지원**:
  - iOS: 홈 화면 추가, 오프라인 지원
  - Android: 완전한 PWA 경험 (설치 프롬프트, 백그라운드 동기화)
- **디바이스**:
  - 모든 스마트폰 (iPhone, Galaxy, 기타 Android)
  - 태블릿 (iPad, Galaxy Tab)
- **해상도**: 반응형 디자인 (320px ~ 1920px)
- **오프라인**: Service Worker로 기본 기능 지원

### 7.3 접근성

- 색각 이상 사용자를 위한 색상 라벨링
- 충분한 터치 영역 크기 (최소 44x44pt)
- 스크린 리더 지원
- **시니어 사용자 고려사항**:
  - 가독성 높은 큰 폰트 사이즈 옵션
  - 고대비 모드 제공
  - 간단하고 직관적인 네비게이션
  - 도움말 및 튜토리얼 쉽게 접근 가능

### 7.4 보안

- **HTTPS 필수**: PWA는 반드시 HTTPS 환경에서 작동
- **Supabase 보안**:
  - Row Level Security (RLS)로 사용자 데이터 격리
  - API Key는 환경 변수로 관리
- **브라우저 저장소 보안**:
  - IndexedDB 데이터는 도메인별 격리
  - 민감한 정보는 저장하지 않음
- **CSP (Content Security Policy)**: XSS 공격 방지
- **사용자 권한**:
  - 카메라 (향후 기능): 브라우저 권한 요청
  - 저장소: 브라우저 자동 관리 (별도 권한 불필요)

## 10. 리스크 및 제약사항

### 10.1 기술적 리스크

- **Canvas 성능 이슈**: 복잡한 템플릿에서 렌더링 성능 저하 가능
  - **완화 방안**:
    - requestAnimationFrame 사용
    - 오프스크린 캔버스 활용
    - 이미지 사전 로딩 및 캐싱
    - 레이어 분리 렌더링
    - WebP 포맷으로 이미지 최적화
- **브라우저 호환성**: 일부 구형 브라우저에서 PWA 기능 제한
  - **완화 방안**:
    - 최신 iOS Safari, Chrome, Samsung Internet 우선 지원
    - 폴리필 적용 (core-js)
    - Progressive Enhancement 전략
    - 시니어 타겟층은 대부분 최신 기기 사용

- **오프라인 저장 용량**: IndexedDB 용량 제한 (브라우저별 5-50MB)
  - **완화 방안**:
    - 이미지 압축 (WebP, 품질 80%)
    - 오래된 작품 자동 클라우드 업로드
    - 사용자에게 저장 공간 상태 표시
    - 평균 작품 크기 1MB 이하로 관리

- **PWA 인식 부족**: 시니어 사용자가 PWA 설치 개념 이해 어려움
  - **완화 방안**:
    - 경로당 워크숍에서 직접 설치 도움
    - 간단한 설치 가이드 동영상 제공
    - "홈 화면에 추가" 안내 팝업
    - QR 코드로 쉬운 접근

- **터치 제스처 충돌**: 줌/팬 제스처와 컬러링 터치 충돌 가능
  - **완화 방안**:
    - 명확한 모드 구분 (컬러링 모드 / 이동 모드)
    - 더블탭 줌 비활성화
    - 핀치 줌만 허용
    - 사용자 설정으로 제스처 커스터마이징

### 10.2 비즈니스 리스크

- **경쟁 앱**: 기존 유사 앱들과의 차별화 필요
  - **완화 방안**: 시니어 친화적 UX, 한국 사용자 맞춤형 콘텐츠

- **수익화**: 초기 무료 모델로 인한 수익 부재
  - **완화 방안**: 사용자 확보 후 점진적 수익화 모델 도입

- **시니어 사용자 접근성**: 디지털 기기에 익숙하지 않은 사용자의 진입 장벽
  - **완화 방안**: 상세한 튜토리얼, 오프라인 안내 자료, 가족 지원 기능

### 10.3 제약사항

- **저작권**: 템플릿 이미지 저작권 확보 필요
- **스토어 정책**: 앱스토어/플레이스토어 가이드라인 준수
- **개인정보**: 사용자 데이터 처리 시 개인정보보호법 준수

### B. 용어 정의

- **Flood Fill**: 영역 자동 채우기 알고리즘
- **MVP**: Minimum Viable Product (최소 기능 제품)
- **KPI**: Key Performance Indicator (핵심 성과 지표)
- **IAP**: In-App Purchase (인앱 구매)
- **시니어 친화 디자인**: 고령 사용자의 시각, 인지, 운동 능력을 고려한 UI/UX 설계
- **디지털 포용성**: 모든 연령대가 기술을 쉽게 사용할 수 있도록 하는 개념
- **햅틱 피드백**: 터치 시 진동으로 피드백을 제공하는 기능
- **PWA (Progressive Web App)**: 웹 기술로 만든 앱으로, 오프라인 작동, 홈 화면 설치 등 네이티브 앱 같은 경험 제공
- **Service Worker**: 백그라운드에서 실행되는 스크립트로, 오프라인 캐싱, 푸시 알림 등 지원
- **Web App Manifest**: PWA 설정 파일 (앱 이름, 아이콘, 테마 색상 등)
- **IndexedDB**: 브라우저 내장 NoSQL 데이터베이스, 대용량 데이터 저장 가능
- **Canvas API**: HTML5의 그래픽 렌더링 API, 2D 그림 그리기에 사용
- **HMR (Hot Module Replacement)**: 코드 수정 시 전체 페이지 새로고침 없이 변경사항만 반영
- **App Router**: Next.js 13+의 새로운 라우팅 시스템, React Server Components 기반
